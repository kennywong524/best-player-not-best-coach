# CentaurBench — reproducibility code

Code and run data for **CentaurBench: Benchmarking LLM Augmentation on Occupational Tasks** (Data Innovation and AI Lab, Haas School of Business, UC Berkeley).

This repository is for reproducibility. It contains the benchmark harness, task configurations, pipeline scripts, the static results dashboard front-end, and the **raw outputs of the three reported runs** under [`results/`](results/) and [`artifacts/`](artifacts/). The large precomputed dashboard data bundle (`dashboard/dashboard-data.json`) is not tracked; rebuild it locally from `results/` (see below).

The benchmark evaluates LLMs in two regimes:

- **Automation** — a model completes a professional task end-to-end.
- **Augmentation** — a focal model writes a *process-only scaffold* that is passed to a fixed lower-capability worker model (GPT-3.5-Turbo), which then produces the final deliverable. Holding the worker constant isolates the value of the assistant's guidance.

Outputs are scored by a panel of LLM judges via **blind pairwise comparisons** with task-specific and general rubrics, under **leave-one-family-out** masking, replicated across **three independent runs**. Built on [Expected Parrot / EDSL](https://www.expectedparrot.com/getting-started) ([docs](https://docs.expectedparrot.com/en/latest)).

- **Interactive results dashboard:** https://kennywong524.github.io/centaur-benchmark/
- **Lab:** [Data Innovation and AI Lab](https://thedatainnovationlab.com/), Haas School of Business, UC Berkeley

## Requirements

- **Python 3.10+** (EDSL on PyPI targets 3.10–3.12)
- An Expected Parrot account with API credits ([getting started](https://www.expectedparrot.com/getting-started))
- Dependencies declared in [`pyproject.toml`](pyproject.toml) (`edsl`, `pandas`, `numpy`, `PyYAML`, `matplotlib`, `seaborn`)

## Install

```bash
python3 -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -e .
```

## Authenticate

Create a `.env` file in the repo root (gitignored):

```bash
EXPECTED_PARROT_API_KEY=your_key_here
EXPECTED_PARROT_URL=https://www.expectedparrot.com
```

By default the harness runs locally through the Expected Parrot API proxy (`CENTAUR_EDSL_REMOTE=0`). Recommended timeouts for long tasks:

```bash
export EDSL_API_TIMEOUT=600 REMOTE_PROXY_TIMEOUT=600 EDSL_MAX_ATTEMPTS=8
```

## Reproduce a run

Each run is identified by a `--run-id`. Tasks, model rosters, and token limits are defined in [`tasks/*.yaml`](tasks/).

```bash
set -a && source .env && set +a
export CENTAUR_EDSL_REMOTE=0 PYTHONPATH=src
RUN_ID=my_run_01

# 1) Generate augmentation + automation outputs for all tasks
python scripts/run_full_pipeline.py --run-id "$RUN_ID" --mode generation

# 2) Audit outputs and scaffolds before judging
python scripts/audit_run.py --run-id "$RUN_ID" --min-chars 250
python scripts/retry_failed.py --run-id "$RUN_ID" --min-chars 250   # optional

# 3) Panel pairwise judging (n-evals 1 for a cheap first pass; 3 matches YAML defaults)
python scripts/run_full_pipeline.py --run-id "$RUN_ID" --mode judge --n-evals 1

# 4) Cross-task rank / win-rate matrices + heatmaps
python scripts/run_full_pipeline.py --run-id "$RUN_ID" --mode summarize
```

Single task at a time via the CLI:

```bash
centaur-benchmark run --task meal_plan --mode all --run-id my_run_01
centaur-benchmark judge --task meal_plan --run my_run_01 --subset both --eval-model gpt-4.1 --n-evals 3
centaur-benchmark summarize --task meal_plan --run my_run_01
```

## Rebuild the dashboard and paper figures

```bash
python scripts/build_dashboard_data.py   # writes dashboard/dashboard-data.json from results/
python scripts/make_paper_figures.py     # writes figures from the dashboard bundle
```

Then open `dashboard/index.html` locally or serve the `dashboard/` directory.

## Included run data

The three reported runs are included so the figures and dashboard can be rebuilt without re-querying any models:

| Run | `run_id` |
|-----|----------|
| Run 1 | `20260610_scaffold_strict_v4` |
| Run 2 | `20260612_fresh_rep1` |
| Run 3 | `20260612_fresh_rep2` |

Output locations:

- `results/<task>/<run_id>/` — raw augmentation/automation outputs, scaffolds, and per-judge CSVs
- `artifacts/cross_task/<run_id>/` — cross-task rank/win-rate matrices and heatmaps

New runs you generate locally land in the same directories; the dashboard data bundle (`dashboard/dashboard-data.json`) is gitignored and rebuilt with the command above.
